## Facebook

InferHub 2.6 is out.

Vector search is very good at "what is this document about" and quite bad at "find the line where error E-4021 is defined". That's not a defect — embeddings encode aboutness, and aboutness is the wrong tool for a literal string. Every RAG system hits this wall eventually.

So retrieval now has three modes: vector, keyword (BM25), and hybrid, which runs both and fuses them by rank (RRF). Plus an opt-in reranking pass that reuses a model already on your fleet, rather than making you deploy another service.

And — the part we care about most — an evaluation harness ships in the same release. "Hybrid improves retrieval" is an empirical claim. We'd rather measure it on your corpus than assert it in a blog post. The harness reports Recall@k, MRR and nDCG per mode, and you can run it on your own data.

Defaults are unchanged. A feature that silently changes your existing results is a regression in a nicer outfit.

Self-hosted, MIT, .NET 10. No new dependencies in this release.

## r/LocalLLaMA (draft)

**InferHub 2.6 — hybrid search + reranking for a self-hosted OpenAI-compatible mesh, with an eval harness in the box**

InferHub is a .NET inference mesh: a GPU-less coordinator on an always-on host exposes an OpenAI/Ollama-compatible API, and GPU nodes dial out to it over SignalR (no port forwarding). RAG has been built in since 2.0.

2.6 goes after retrieval quality. Three per-request modes (`vector` / `keyword` / `hybrid`), keyword being BM25 (in-memory index under the local store, Postgres full-text under pgvector), hybrid fusing both by Reciprocal Rank Fusion. Plus an opt-in LLM reranker that reuses a model already on the fleet instead of making you stand up a cross-encoder service.

The part I care about: it ships with an eval harness. "Hybrid improves retrieval" is an empirical claim, and it depends on your corpus — mostly prose sees a smaller gain than technical docs full of identifiers. The harness runs a golden set in every mode and prints Recall@k / MRR / nDCG. One honest caveat baked into its README: a golden set generated by the same model you're evaluating is a mirror, not evidence.

Defaults are unchanged — no headers, no config, identical to 2.5. MIT, no new dependencies.
